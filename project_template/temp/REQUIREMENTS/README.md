# Requirements

这个目录给用户放需求文档。

用户也可以直接在聊天里描述需求，不强制必须先写文件。

如果需求比较长、需要多人确认、后续会反复迭代，建议放到这个目录，方便 AI 和用户一起追踪。

## 使用规则

1. 一个需求尽量一个 Markdown 文件
2. 文件内容尽量写清楚业务目标、页面、角色、流程、限制条件
3. 如果本目录有真实需求文档，AI 会优先读取这里，再开始设计
4. 示例文件只用于参考，不能默认当成真实开发任务

## 建议先看

1. `requirement_template.md`
2. `requirement_example_order.md`

## 看完示例后的建议

用户理解怎么写需求以后，建议直接删除示例文件，只保留自己真正要用的需求文档。
