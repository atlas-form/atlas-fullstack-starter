# Fullstack Workflow

这个文件定义 AI 在这个仓库里的标准工作流。

## 标准流程

1. 用户把需求文档放进 `requirements/`
2. AI 阅读需求文档并整理问题
3. AI 先写 `development_docs/backend/` 下的后端开发文档
4. 后端开发文档必须包含数据库表结构、接口、鉴权、业务流程
5. 用户确认后端开发文档
6. AI 写 `api_docs/` 下的接口文档
7. AI 再写 `development_docs/frontend/` 下的前端开发文档
8. 用户确认
9. AI 开始后端开发
10. AI 开始前端开发和联调

## 后端文档要求

后端开发文档至少要回答这些问题：

1. 需要哪些表
2. 每张表有哪些字段
3. 字段类型、约束、索引是什么
4. JWT 鉴权怎么接入
5. 要提供哪些接口
6. 每个接口的请求和响应是什么
7. 错误码和边界情况是什么

## 前端文档要求

前端开发文档不能先于后端接口设计出现。

前端开发文档至少要回答这些问题：

1. 页面或模块有哪些
2. 每个页面调用哪个接口
3. 请求参数和响应数据如何映射到页面
4. 登录态和用户标识如何使用
5. 异常状态和空状态怎么处理

## API 文档要求

`api_docs/` 里的文档是给前端和后端共同使用的。

每个接口文档至少要写清楚：

1. 路由
2. 方法
3. 是否需要 JWT
4. 请求参数
5. 响应结构
6. 错误响应
7. 字段说明

## 禁止事项

1. 不允许跳过 `requirements/` 直接凭空开发
2. 不允许先写前端再倒推接口
3. 不允许后端还没定 API 就开始前端联调
4. 不允许用户未确认文档就直接进入正式开发
