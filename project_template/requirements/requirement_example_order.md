# Requirement Example: Order Query

这个文件是一个给用户参考的需求示例。

## 重要说明

1. 这个文件只是示例，不是默认要开发的功能
2. AI 不可以因为看到了这个文件，就直接开始实现这个功能
3. 只有当用户明确说“按这个示例开发”或明确引用这个文件时，AI 才能把它当成真实需求
4. 用户理解这个示例以后，建议直接删除这个文件，避免后续误用

## 需求名称

订单列表与订单详情查询

## 业务目标

用户希望在系统里查看自己提交过的订单，并能进入详情页查看订单内容、状态和创建时间。

## 使用角色

普通登录用户

## 业务流程

1. 用户登录系统
2. 进入“我的订单”页面
3. 系统展示当前用户的订单列表
4. 用户点击某一条订单
5. 系统进入订单详情页并展示详细信息

## 页面或入口

1. 用户中心里的“我的订单”入口
2. 订单列表页
3. 订单详情页

## 需要保存的数据

1. 订单编号
2. 用户 ID
3. 订单状态
4. 订单总金额
5. 订单创建时间
6. 订单里的商品信息

## 规则和限制

1. 用户只能查看自己的订单
2. 未登录用户不能访问订单相关页面
3. 订单列表支持按状态筛选
4. 订单列表需要分页
5. 订单详情页必须展示订单状态和商品明细

## 你希望的结果

1. 用户可以在前端看到自己的订单列表
2. 用户可以点击进入订单详情页
3. 系统接口返回的数据足够前端直接展示
4. 整个流程需要基于已有 JWT 登录态工作
