# User Start

这个文件是给普通用户看的开场提示词模板。

AI 的入口不是本文件。AI 必须从根目录 `AGENTS.md` 开始阅读。

## 用户第一次可以直接发给 AI 的话

```text
请先阅读当前项目根目录 AGENTS.md，再按 AGENTS.md 的要求阅读相关协议和项目文档，然后再开始工作。

在你确认自己已经读完这些协议之前，不要直接回答我的业务问题，也不要直接开始写代码。

你现在的工作目录就是这个前后端一体项目的根目录。

请按下面规则工作：
1. 先检查前端和后端环境是否完整，缺什么你自己安装或修复
2. 如果数据库初始化失败，你先自己排查；实在不行再告诉我需要一个可用数据库链接
3. 不要直接开始写业务代码，先让我描述需求
4. 我可能会把需求文档放到 REQUIREMENTS/ 目录，也可能直接在聊天里描述需求
5. 你要先整理我的需求摘要和待确认问题，再从服务端设计开始
6. 先写 DEVELOPMENT_DOCS/backend/ 下的后端开发文档和数据库表结构给我确认
7. 后端 API 设计确认后，再写 DEVELOPMENT_DOCS/frontend/ 和 API_DOCS/ 给前端使用
8. 我确认文档后，你再正式开发
9. 如果后端接口有新增或修改，必须同步更新根目录 API_DOCS/
10. 如果过程中报错，优先自己处理，不要把原始报错直接甩给我
```

## 用户需要知道的最少规则

1. 需求可以放进 `REQUIREMENTS/`，也可以直接在聊天里描述
2. 每次和 AI 开始新对话，都让 AI 从 `AGENTS.md` 开始阅读
3. AI 必须先写后端开发文档，不能跳过设计直接开发
4. 后端 API 明确以后，AI 才能写前端开发文档
5. 前后端联调时，以根目录 `API_DOCS/` 里的接口文档为准
