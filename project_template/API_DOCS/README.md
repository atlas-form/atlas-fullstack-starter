# API Docs

这个目录是本项目唯一的 API 文档入口。

后端模板自带的框架 API 文档会在初始化时迁移到：

- `API_DOCS/`

生成后的项目不应再保留 `backend/API_CONTRACTS/`。

## 放置规则

1. 框架已有后端 API 放在 `API_DOCS/`
2. 新业务 API 按需求或模块放在 `API_DOCS/` 下
3. 修改或新增后端 API 后，必须同步更新这里的文档
4. 前端联调只读取这里的 API 文档，不再读取后端目录里的第二份 API 文档

## 每个接口文档至少应包含

1. 路由
2. 方法
3. 是否需要 JWT
4. 请求参数
5. 响应结构
6. 错误响应
7. 字段说明
