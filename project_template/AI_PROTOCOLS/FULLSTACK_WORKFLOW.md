# Fullstack Workflow

这个文件定义 AI 在这个仓库里的标准工作流。

## 标准流程

0. AI 先阅读根目录协议，再开始回答用户问题
1. 用户通过 `REQUIREMENTS/` 或聊天消息提供需求
2. AI 阅读需求文档，或整理聊天中的需求摘要和待确认问题
3. AI 先写 `DEVELOPMENT_DOCS/backend/` 下的后端开发文档
4. 后端开发文档必须包含数据库表结构、接口、鉴权、业务流程
5. 用户确认后端开发文档
6. AI 写 `API_DOCS/` 下的前后端协作接口文档
7. AI 再写 `DEVELOPMENT_DOCS/frontend/` 下的前端开发文档
8. 用户确认
9. AI 开始后端开发
10. AI 开始前端开发和联调
11. 如果后端接口有新增或修改，AI 同步更新根目录 `API_DOCS/`

## 后端文档要求

后端开发文档至少要回答这些问题：

1. 需要哪些表
2. 每张表有哪些字段
3. 字段类型、约束、索引是什么
4. JWT 鉴权怎么接入
5. 要提供哪些接口
6. 每个接口的请求和响应是什么
7. 错误码和边界情况是什么

## 前端文档要求

前端开发文档不能先于后端接口设计出现。

前端开发文档至少要回答这些问题：

1. 页面或模块有哪些
2. 每个页面调用哪个接口
3. 请求参数和响应数据如何映射到页面
4. 登录态和用户标识如何使用
5. 异常状态和空状态怎么处理

## API 文档要求

`API_DOCS/` 是本项目唯一的 API 文档入口。

后端模板自带的框架 API 文档会在初始化时迁移到 `API_DOCS/`。

放置规则：

1. 框架已有后端 API 放在 `API_DOCS/`
2. 新业务 API 按需求或模块放在 `API_DOCS/` 下
3. 后端接口实现或修改后，必须同步更新根目录 `API_DOCS/`
4. 前后端联调只读取根目录 `API_DOCS/`

每个接口文档至少要写清楚：

1. 路由
2. 方法
3. 是否需要 JWT
4. 请求参数
5. 响应结构
6. 错误响应
7. 字段说明

## 禁止事项

1. 不允许在没有需求文档、也没有聊天需求描述的情况下凭空开发
2. 不允许先写前端再倒推接口
3. 不允许后端还没定 API 就开始前端联调
4. 不允许用户未确认文档就直接进入正式开发
5. 不允许未阅读根目录协议就开始回答用户问题
6. 不允许回答用户时脱离本项目工作流
7. 不允许在 `backend/` 下维护第二份 API 文档目录
