# AI Start

这个文件是给普通用户直接发给 AI 的。

## 用户第一次可以直接发给 AI 的话

```text
请先阅读当前项目根目录 README.md、AI_START.md、ai_protocols/、requirements/、development_docs/、api_docs/，再开始工作。

在你确认自己已经读完这些协议之前，不要直接回答我的业务问题，也不要直接开始写代码。

你现在的工作目录就是这个前后端一体项目的根目录。

请按下面规则工作：
1. 先检查前端和后端环境是否完整，缺什么你自己安装或修复
2. 如果数据库初始化失败，你先自己排查；实在不行再告诉我需要一个可用数据库链接
3. 不要直接开始写业务代码，先让我描述需求
4. 我会把需求文档放到 requirements/ 目录
5. 你必须先从服务端设计开始
6. 先写 development_docs/backend/ 下的后端开发文档和数据库表结构给我确认
7. 后端 API 设计确认后，再写 development_docs/frontend/ 和 api_docs/ 给前端使用
8. 我确认文档后，你再正式开发
9. 如果过程中报错，优先自己处理，不要把原始报错直接甩给我
```

## 用户需要知道的最少规则

1. 需求文档放进 `requirements/`
2. 每次和 AI 开始新对话，先让 AI 读完根目录协议再继续
3. AI 必须先写后端开发文档，不能跳过设计直接开发
4. 后端 API 明确以后，AI 才能写前端开发文档
5. 前后端联调时，优先以 `api_docs/` 里的接口文档为准
